## Ouvrir une case du calendrier

Pour ouvrir votre case quotidienne du calendrier de l'avent, utilisez la commande \</calendrier> pendant le mois de décembre.

::hint{ type="info" }
  Le calendrier de l'avent est accessible **du 1er au 24 décembre** uniquement. En dehors de cette période, la commande affichera un aperçu du calendrier.
::

::hint{ type="success" }
  Chaque membre peut ouvrir **une case par jour** correspondant à la date du jour. Les cases déjà ouvertes sont marquées d'une croix sur le calendrier.
::

## Les surprises quotidiennes

Le calendrier de l'avent permet d'offrir différents types de surprises à vos membres :

| Type | Description | Exemple |
|------|-------------|---------|
| **Rôle permanent** | Attribution d'un rôle Discord définitif | Rôle "Noël 2024" le 25 décembre |
| **Rôle temporaire** | Attribution d'un rôle pour une durée limitée | Rôle "Lutin du jour" pendant 24h |
| **Expérience** | Quantité d'expérience du [système de niveaux](/docs/modules/niveaux) | 500 XP le 10 décembre |
| **Argent** | Quantité d'argent virtuel du [système d'économie](/docs/modules/economie) | 1000 pièces le 15 décembre |
| **Objet d'inventaire** | Un ou plusieurs [objets d'inventaire](/docs/modules/inventaire) | 2x "Flocon magique" le 20 décembre |
| **Surprise personnalisée** | Une récompense que vous remettez manuellement | Code promo, goodies physiques, etc. |

::hint{ type="success" }
  Vous pouvez ajouter **2 surprises par case**, DraftBot en choisira une **aléatoirement**. Les serveurs [premium](/premium) peuvent en ajouter jusqu'à **5 par case** !
::

## Récompenses d'ouvertures consécutives

Récompensez la régularité de vos membres avec le système d'ouvertures consécutives ! Les membres qui ouvrent leur case chaque jour sans interruption peuvent recevoir des récompenses bonus.

### Modes de gain

Deux modes de gain sont disponibles pour chaque récompense :

| Mode | Description | Exemple |
|------|-------------|---------|
| **Mode unique** | La récompense est obtenue une seule fois lorsque l'objectif est atteint | Rôle "Fidèle" à 7 ouvertures consécutives |
| **Mode multiple** | La récompense est obtenue à chaque fois que l'objectif est atteint de manière répétée | 100 pièces toutes les 5 ouvertures consécutives |

::hint{ type="info" }
  **Réobtensible** : Pour les récompenses en mode unique, vous pouvez activer l'option "réobtensible" pour permettre aux membres de gagner à nouveau la récompense s'ils cassent leur série puis atteignent de nouveau l'objectif.
::

### Comptabilisation des ouvertures

Vous pouvez choisir la méthode de comptage des ouvertures consécutives :

- **Toutes années confondues** : Les ouvertures des années précédentes sont comptabilisées
- **Année en cours uniquement** : Seules les ouvertures de l'année actuelle comptent (utile pour les objectifs ≤ 24 jours)

::hint{ type="warning" }
  La série d'ouvertures consécutives se réinitialise si un membre **oublie une journée**. Par exemple, s'il ouvre les cases des 1er, 2 et 4 décembre (sans ouvrir celle du 3), sa série redémarre à 1.
::

::hint{ type="success" }
  Les serveurs peuvent créer jusqu'à **3 récompenses** d'ouvertures consécutives. Les serveurs [premium](/premium) peuvent en créer jusqu'à **10** !
::

## Personnalisation

### Message personnalisé

Vous pouvez personnaliser le message affiché lorsqu'un membre ouvre une case du calendrier. Les variables suivantes sont disponibles :

| Variable | Description | Exemple |
|----------|-------------|---------|
| `{calendar.day}` | Numéro du jour de la case ouverte | 15 |

*En plus des [autres variables](docs/annexes/variables) déjà disponibles globalement !*

### Message d'ouvertures consécutives

Un message séparé peut être configuré pour les récompenses d'ouvertures consécutives, avec les variables supplémentaires suivantes :

| Variable | Description | Exemple |
|----------|-------------|---------|
| `{calendar.streak.count}` | Nombre d'ouvertures consécutives actuelles | 7 |
| `{calendar.streak.record}` | Record d'ouvertures consécutives du membre | 15 |
| `{calendar.streak.reward}` | Nom de la récompense obtenue | Rôle "Fidèle" |

*En plus des [autres variables](docs/annexes/variables) déjà disponibles globalement !*

### Image de fond personnalisée

Les serveurs [premium](/premium) peuvent définir une image de fond personnalisée pour le calendrier.

::hint{ type="success" }
  **Dimensions recommandées** : 1740 x 1260 pixels. Vous pouvez également ajouter un filtre gris semi-transparent pour améliorer la lisibilité.
::

## Comportements particuliers

### Surprises avec rôles uniques

Lorsqu'un rôle est configuré comme "unique" :
- Si le membre possède déjà un rôle unique d'une case précédente, ce rôle sera **automatiquement retiré**
- Seul le rôle unique le plus récent est conservé

### Surprises personnalisées

Lorsqu'un membre obtient une surprise personnalisée :
- Un **message privé** est envoyé au créateur de la surprise avec les informations du membre
- Le créateur peut alors remettre la récompense manuellement

::hint{ type="warning" }
  Assurez-vous que vos messages privés sont ouverts pour recevoir les notifications ! Si vos MP sont fermés ou si vous avez quitté le serveur, la surprise ne pourra pas être remise.
::

### Limitations de stock d'objets

Les membres ne peuvent pas dépasser la limite de **1 000 000 exemplaires** d'un même objet d'inventaire. Si cette limite est atteinte, l'objet ne sera pas ajouté.

### Permissions requises pour les rôles

Pour attribuer des rôles, DraftBot doit disposer de :
- La permission **"Gérer les rôles"**
- Un rôle **plus haut** dans la hiérarchie que les rôles à attribuer

Si ces conditions ne sont pas remplies, le membre sera notifié mais ne recevra pas le rôle.

## Système de séries consécutives

Le système d'ouvertures consécutives fonctionne de la manière suivante :

### Calcul de la série

- Chaque jour où un membre ouvre sa case **sans interruption**, sa série augmente de 1
- Si un membre **saute un jour**, sa série se réinitialise à 1
- La série peut se poursuivre **d'une année sur l'autre** (la case du 24 décembre suivie de la case du 1er janvier maintient la série)

**Exemple :**
- Ouverture des cases du 1er, 2, 3, 4 décembre → Série de 4
- Pas d'ouverture le 5 décembre
- Ouverture de la case du 6 décembre → Série de 1 (réinitialisée)

### Comptabilisation par année

Pour les récompenses configurées avec "année en cours uniquement" :
- Seules les ouvertures de décembre de l'année actuelle sont comptées
- Utile pour créer des objectifs sur la période du calendrier (objectifs ≤ 24 jours)
